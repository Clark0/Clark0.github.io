FPN

Posted on 2020-01-23

FPN

Architecture

Leverage a ConvNet’s pyramidal feature hierarchy (semantics from low to high levels) to build a feature pyramid with high-level semantics through out.

Take a single-scale image of any size, and output proportionally sized feature maps at multiple levels.

Bottom-up pathway

Feed-forward computation of backbone ConvNet. During feed-forward, layers output maps of the same size are grouped as network $stage$. Each stage defines one pyramid level.

For ResNets, uses feature maps output by each stage’s last residual block, denotes ${C2, C3, C4, C5}$ for conv2, conv3, conv4, conv5 outputs. They have strides of {4, 8, 16, 32} pixals input with respect to input image

Top-down pathway and lateral connections

Feature maps from higher pyramid levels upsampling (nearest neighbor upsampling)

  • higher pyramid levels has stronger semantic feature

  • lower pyramid levels has more accurately localized activations

Apply FPN to RPN

Apply network head to each level of FPN.

Assign anchors with the same scale to each level. No need to have multi-scale anchors on a specific level.

Define anchors to have scales ${32, 64, 128, 256, 512}$ on ${P2, P3, P4, P5, P6}$. Each anchor has ratio of ${1:2, 1:1, 2:1}$
